5
Why it beats a RAG chatbot — and the drawing tools

The question that separates us:

"P-101 keeps tripping on high temperature — what feeds it, has this happened before, and what does the manual say to check?"
  • Graph traces CT-101 → STR-101 → P-101 and lights the feed path on drawing D-CW-104
  • Text finds both monsoon-2025 trip work orders — root cause: fouled cooler
  • Visual retrieves the scanned, skewed, stamped OEM troubleshooting page by appearance — no OCR ever ran
  • One streamed answer, every sentence cited, confidence scored, source pages one tap away

A connectivity-only P&ID tool cannot answer this. A text-RAG cannot show it. The fusion is the product.

6
The anti-knowledge-cliff engine

It connects dots no individual can — then acts before the repeat

RCA across the fleet

Five monsoon seal failures on P-101 (2019→2025) + the retiring engineer's analysis + strainer ΔP inspections → causal chain, and the callout: "same pattern on P-103 and P-107 — recommend fleet-wide seal review."

Foresight, not hindsight

Tomorrow's TK-401 confined-space permit plans a 2-hour purge. SMRITI matches it against near-misses NM-2019-07 / NM-2022-31 (same tank, monsoon, short purge, H2S re-release) and pushes the warning before the crew enters.

India-first compliance

18 real clauses — OISD-116/129/132/105, Factories Act §§31–40, SMPV(U) 2016, Petroleum Rules 2002 — mapped to equipment, evaluated against actual records, one-click audit evidence pack. It even catches the plant's own inspection plan contradicting OISD-129's annual requirement.

Technical excellence — engineering maturity most demos skip

We measure it, live

  • Golden set: 33 domain-expert Q&As — multi-hop, visual, RCA, compliance, temporal, and honest out-of-corpus questions
  • LLM-judged faithfulness (claims ⊆ evidence), citation correctness (does the cited page support the sentence?), expected-point coverage
  • Baseline comparison: same corpus, same answer model, vanilla single-vector RAG — the delta is the fabric's contribution, shown in-app
  • "Insufficient evidence" is a first-class answer — trust is a feature
  • Every tool call logged; provenance object on every node and edge → audit-admissible
